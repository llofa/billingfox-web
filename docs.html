API Documentation

Getting Started

​​Implement our API by setting up actions within your service and then assigning worth to those actions via spend​ calls to BillingFox.

Open this page, watch that video, call the other API. Whatever. Just correlate those actions to spending a user's account credits.

2 credits here, 10 credits there, 0.073 credits everywhere. You decide.

When a user's balance starts getting low and hits your alert level we'll send them an email invoice to refill their account. Once they fill out that invoice we'll credit their balance, subtract the fees and pass on the rest to you.

​​As a concrete example, if a user were to fill out a $25 invoice

  • Stripe would take out 2.9% + $0.3​ = $1.03
  • BillingFox would take out 5% = $1.25
  • The user would receive 2,500 credits
  • Leaving you with $22.72

How does it work?

BillingFox operates on three core services. Stellar, Stripe and the BillingFox API.

Stellar - acts as the financial underlayer for the platform.

Providing a tokenization engine much like Bitcoin or Ethereum.

Within their network we create and maintain a network of accounts which give and receive a token unique to BillingFox. Stellar acts as the financial ledger keeping track of who has what and how much.

Stripe - acts as the currency platform

Allowing BillingFox to move from tokens, cryptocurrencies and computers to cold hard cash in your bank account.

The Stripe Connect product is a top notch tool allowing BillingFox to safely, securely and in a controlled, maintained environment move money from your users to you. All without you ever having to really do much of anything beyond creating an initial connection.

BillingFox API - is the last piece of the puzzle

While it might sound tricky or feel like black magic at times. It’s actually just simple, smart and straightforward. You give us a user and then begin spending their credit balance.

What’s the ratio of $$ to user credit?

Simple. If a user clears a $10 invoice we’ll credit them 1,000 credits. 1 credit = 1¢ Use this information to decide how much you want to spend per call when it comes time to make those calls. This is also obviously helpful for deciding on your account settings page what to set the initial credit and alert level at.

What does BillingFox cost?

BillingFox has a fixed fee of 5%. However this is added on top of Stripe’s fee of 2.9% + 30¢ per transaction. So for example if a user where to clear that $10 invoice, Stripe would take 59¢, BillingFox would take 50¢, leaving you with $8.91.

Authentication

Once you've created and signed into your BillingFox account you'll be presented with a screen containing your API key.

You'll simply need to include that token as an Authorization header with the Bearer prefix.

Once you've got that set you're ready to run with the BillingFox API!

Errors

All errors are in the format:

{ // 400
	"message": "Unauthorized. Access token is invalid",
	"status": "error"
}
						
{ // 402
	"message": "Insufficient available balance to spend 5 credits",
	"status": "process"
}

POST Identify

​​Create user's within the BillingFox ecosystem whose credits you can then begin to spend.

POST /identify
						
curl --request POST \
	--url https://test.billingfox.com/api/identify \
	--header 'accept: application/json' \
	--header 'authorization: Bearer YOUR API KEY HERE' \
	--form user=user_abc123 \
	--form email=demo@billingfox.com
						
{ 	// The ID for this user
	user: "user_abc123",
	// A valid email address for this user, so we can send them invoices
	email: "demo@billingfox.com"
}
						
{ // 200 Success
	"message": "User successfully created/updated",
	"status": "success"
}
						
{ // 400 Error
	"message": "Match error: Failed Match.Where validation in field email",
	"status": "error"
}

GET Identify

Retrieve a user’s data. Email and balance.

GET /identify
							
curl --request GET \
	--url https://test.billingfox.com/api/identify?user=user_abc_123 \
	--header 'accept: application/json' \
	--header 'authorization: Bearer YOUR API KEY HERE'
							
{
	// ID for the user you'd like to retrieve
	user: "user_abc_123"
}
							
{ // 200 Success
	"user": {
		"key": "3vSdThRP",
		"email": "demo@billingfox.com",
		"balances": {
		"available": "95.0000000",
		"current": "100.0000000"
		},
		"link": "https://billlingfox.com/invoice/abc_123_link_to_invoice"
	},
	"status": "success"
}
							
{ // 400 Error
	"message": "Match error: Failed Match.Where validation in field user",
	"status": "error"
}

POST Spend

Spend a user's account credit balance.

POST /spend
						
curl --request POST \
	--url https://test.billingfox.com/api/spend \
	--header 'accept: application/json' \
	--header 'authorization: Bearer YOUR API KEY HERE' \
	--form amount=1.0 \
	--form user=user_abc123
						
{
	// Whose fuel would you like to burn?
	user: "user_abc123",
	// How much fuel would you like to burn?
	amount: 1.0,
	// Any description text you'd like to append to this spend call
	description: "A helpful description for this spend call"
}
						
{ // 200 Success
	"message": "5 credits were spent",
	"balances": {
		"available": "95.0000000",
		"current": "100.0000000"
	},
	"status": "success"
}
						
{ // 400 Error
	"message": "Match error: Failed Match.Where validation in field user",
	"status": "error"
}
						
{ // 402 Insufficent
	"message": "Insufficient available balance to spend 2 credits",
	"link": "https://billingfox.com/invoice/abc_123_link_to_invoice",
	"balances": {
		"available": "2.0000000",
		"current": "2.0000000"
	},
	"status": "process"
}

GET Spend

Retrieve an array of spends for a given user.

GET /spend								
							
curl --request GET \
	--url https://test.billingfox.com/api/spends?user=user_abc_123>e=2017-08-01<e=2017-08-31 \
	--header 'accept: application/json' \
	--header 'authorization: Bearer YOUR API KEY HERE'
							
{
	// User id you'd like to retrieve the spends for
	user: "user_abc_123",
	// Greater than or equal to date string for limiting the array of spends
	gte: "2017-08-01"
	// Less than or equal to date string for limiting the array of spends
	lte: "2017-08-31"
}
							
{ // 200 Success
	"spends": [{
		"_id": "spend_abc_123",
		"amount": 2,
		"description": "Some meta data",
		"spent_at": "2017-08-04T15:22:55.833Z"
	}, ...],
	"status": "success"
}
							
{ // 400 Error
	"message": "Match error: Failed Match.Where validation in field user",
	"status": "error"
}
